# 课程小方格高亮缺失和节假日联动修复计划

## Bug 描述

1. **很多课不显示下节课高亮**：当前周该课的所有课都上完后，不会查找下一周的课来高亮
2. **没有和节假日联动**：节假日（如劳动节 5/1-5/5）期间的课仍然有方格，应该跳过

## 根因分析

### Bug 1：高亮缺失

当前 `nextDotIndex` 逻辑（第242-257行）只查找 `d.week === weekNum` 的方格。如果当前周该课的所有课都已结束（比如周五晚上，所有课都上完了），`nextDotIndex` 会保持 -1，不会向前查找下一周。

**修复**：改为查找 `d.week >= weekNum` 的第一个未结束的课，而非仅限当前周。

### Bug 2：没有节假日联动

`dots` 数组（第230-238行）为每周每节课都生成了方格，没有检查该天是否是节假日或补课日。

`schedule.ts` 中的 `getWeekCourses` 已经有完整的节假日过滤逻辑，但课程页面没有使用。

**修复**：在生成 `dots` 时，对每个方格检查其对应日期是否是节假日。如果是节假日且有补课，则调整 `day_of_week`；如果是节假日且无补课，则跳过该方格。

## 修复方案

### 修改文件

`course-schedule/src/app/courses/page.tsx`

### 步骤 1：在 dots 生成中加入节假日过滤

导入 `isHoliday` 和 `getMakeupInfo`：

```js
import { getWeekNumber, getSemesterConfig, getDayDate, getCurrentPeriod, PERIOD_TIMES, isHoliday, getMakeupInfo } from '@/lib/semester'
```

修改 dots 生成逻辑，对每个方格检查节假日：

```js
const dots = []
for (let w = 1; w <= totalWeeks; w++) {
  const wOdd = w % 2 === 1
  for (const s of allCourseSchedules) {
    if (s.week_type === 'all' || (s.week_type === 'odd' && wOdd) || (s.week_type === 'even' && !wOdd)) {
      const dotDate = getDayDate(w, s.day_of_week)
      const holiday = isHoliday(dotDate)
      const makeup = getMakeupInfo(dotDate)
      if (holiday && !makeup) continue
      dots.push({ week: w, schedule: s })
    }
  }
}
```

注意：补课日的处理比较复杂。补课日 `makeup` 的 `replacesDayOfWeek` 表示补的是周几的课。如果某天是补课日，它补的是 `replacesDayOfWeek` 的课，而不是本来那天的课。但当前 dots 的生成是按 schedule 的 `day_of_week` 来算日期的，补课日是另一个日期。

更精确的处理：对于每个方格，先算出该课的日期 `dotDate`，然后：
- 如果 `dotDate` 是节假日且没有补课 → 跳过（不上课）
- 如果 `dotDate` 是节假日但有补课 → 需要检查补的是否是当前 schedule 的 `day_of_week`
- 如果 `dotDate` 不是节假日 → 正常显示

但补课日本身也需要生成方格。补课日是某个非工作日要补周几的课。比如 5/9（周六）补周二（day_of_week=2）的课。那么 5/9 这天，day_of_week=2 的课应该有一个方格。

完整逻辑：
1. 遍历每周的每一天（周一到周五）
2. 对于每一天，判断是否是节假日/补课日
3. 如果是补课日，该天要上 `replacesDayOfWeek` 的课
4. 如果是正常日，该天上本来那天的课
5. 如果是纯节假日，该天不上课

为了简化实现，可以复用 `getWeekCourses` 的逻辑。但 `getWeekCourses` 是泛型函数，返回的是课表项，我们需要的是带周次信息的 dots。

更好的方案：直接在 dots 生成中，对每周的每一天判断是否活跃，然后匹配对应的 schedule：

```js
const dots = []
for (let w = 1; w <= totalWeeks; w++) {
  const wOdd = w % 2 === 1
  const range = getWeekDateRange(w)
  const activeDays = new Map<number, boolean>() // day_of_week -> is active
  for (let i = 0; i < 7; i++) {
    const d = new Date(range.start)
    d.setDate(range.start.getDate() + i)
    const dow = d.getDay() || 7
    const holiday = isHoliday(d)
    const makeup = getMakeupInfo(d)
    if (makeup) {
      activeDays.set(makeup.replacesDayOfWeek, true)
    } else if (!holiday) {
      activeDays.set(dow, true)
    }
  }
  for (const s of allCourseSchedules) {
    if (s.week_type === 'all' || (s.week_type === 'odd' && wOdd) || (s.week_type === 'even' && !wOdd)) {
      if (activeDays.has(s.day_of_week)) {
        dots.push({ week: w, schedule: s })
      }
    }
  }
}
```

### 步骤 2：修复 nextDotIndex 逻辑

将查找范围从 `d.week === weekNum` 改为 `d.week >= weekNum`：

```js
let nextDotIndex = -1
if (weekNum > 0 && currentDayOfWeek > 0) {
  for (let di = 0; di < dots.length; di++) {
    const d = dots[di]
    if (d.week < weekNum) continue
    const s = d.schedule
    if (d.week > weekNum || s.day_of_week > currentDayOfWeek) {
      nextDotIndex = di
      break
    }
    if (s.day_of_week === currentDayOfWeek) {
      if (currentPeriod === null || currentPeriod <= s.end_period) {
        nextDotIndex = di
        break
      }
    }
  }
}
```
